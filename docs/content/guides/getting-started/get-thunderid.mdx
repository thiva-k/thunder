# Get <ProductName />

<ProductName /> is available in multiple formats to suit your deployment needs. Choose the option that works best for you.

<Tabs>
<TabItem value="release" label="Release Artifact" default>

Get the latest release of <ProductName /> and run it locally on your machine.

## Prerequisites

- Your operating system and architecture (macOS, Linux, or Windows)
- Terminal or command prompt access

<Stepper stepNode="h2" as="h3">

## Download the Distribution

Download `{{productSlug}}-<version>-<os>-<arch>.zip` from the <RepoLink path="/releases/latest">latest release</RepoLink> for your operating system and architecture.

:::info Example
If you're using a MacOS machine with Apple Silicon (ARM64) processor, download `{{productSlug}}-<version>-macos-arm64.zip`
:::

## Extract the Archive

Unzip the downloaded file:

```bash
unzip {{productSlug}}-<version>-<os>-<arch>.zip
```

Navigate to the extracted directory:

```bash
cd {{productSlug}}-<version>-<os>-<arch>/
```

## Setup <ProductName />

Initialize <ProductName /> with the default configurations and data before starting the server for the first time.

<Tabs groupId="os">
<TabItem value="unix" label="Linux / macOS">

```bash
./setup.sh
```

</TabItem>
<TabItem value="windows" label="Windows">

```powershell
.\setup.ps1
```

</TabItem>
</Tabs>

## Start <ProductName />

Launch the <ProductName /> server:

<Tabs groupId="os">
<TabItem value="unix" label="Linux / macOS">

```bash
./start.sh
```

</TabItem>
<TabItem value="windows" label="Windows">

```powershell
.\start.ps1
```

</TabItem>
</Tabs>

:::tip Success
<ProductName /> will start on **https://localhost:8090**
:::

</Stepper>

</TabItem>

<TabItem value="docker" label="Docker Compose">

The quickest way to get <ProductName /> up and running with all dependencies configured.

## Prerequisites

- Docker and Docker Compose installed on your machine
- Terminal or command prompt access

<Stepper stepNode="h2" as="h3">

## Download Docker Compose File

Download the `docker-compose.yml` file from the <ProductName /> repository:

{/* The release workflow keeps this versioned URL in sync. */}
```bash
curl -o docker-compose.yml https://raw.githubusercontent.com/thunder-id/thunderid/v0.39.0/install/quick-start/docker-compose.yml
```

## Start <ProductName />

Run the following command in the directory where you downloaded the `docker-compose.yml` file:

```bash
docker compose up
```

:::note
This command will automatically:
- Initialize the database
- Run the setup process
- Start the <ProductName /> server
:::


:::tip Success
<ProductName /> will start on **https://localhost:8090**
:::

</Stepper>

</TabItem>
</Tabs>
