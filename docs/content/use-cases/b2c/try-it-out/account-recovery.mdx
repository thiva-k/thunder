# Account Recovery

In this walkthrough, John has forgotten his Wayfinder password. He selects **Forgot password?** on the sign-in screen, receives a recovery email, opens the link, and sets a new password. <ProductName /> runs the recovery flow end-to-end and returns him to Wayfinder ready to sign in with the new credentials.

:::note Prerequisites
Complete [Setup](../#setup) before starting this walkthrough.
:::

:::info Background
[Account Recovery](../../customer-identity#account-recovery) covers the requirements story behind this use case.
:::

## Walkthrough Setup

1. **Confirm SMTP is configured.** Recovery emails are delivered through the configured SMTP server.

2. **Add a recovery flow.** Pick the tab that matches the path you used in Setup.

<Tabs>
<TabItem value="declarative" label="Quick Start" default>

The bundle already includes the `wayfinder-recovery-flow` attached to `WAYFINDER`. No further action is needed.

</TabItem>
<TabItem value="build" label="Configure It Yourself">

1. Create a recovery flow through the flows API. The flow follows this sequence of steps:

    - Prompts the user for a username.
    - Identifies the user.
    - Generates a recovery token.
    - Sends the recovery email.
    - Verifies the token from the email link.
    - Lands the user on a **Set new password** screen and updates the credential.

    Save the JSON below as `wayfinder-recovery-flow.json` and POST it to the flows API:

    ```bash
    curl -k -X POST https://localhost:8090/flows \
      -H "Authorization: Bearer $TOKEN" \
      -H "Content-Type: application/json" \
      -d @wayfinder-recovery-flow.json
    ```

## Pick Your Pattern

<details open>
  <summary>Redirect-based</summary>

In the redirect-based pattern, the consumer app sends the user to <ProductName /> for the entire recovery experience. The Wayfinder web frontend is configured exactly this way: the **Sign in** button triggers an OIDC redirect to <ProductName />. From there, the user picks **Forgot password?**, completes the recovery flow, and returns to Wayfinder to sign in with the new password.

**Try the Use Case**

1. Open [http://localhost:5173](http://localhost:5173) and select **Sign in**. The browser navigates to <ProductName />.
2. On the <ProductName /> sign-in page, select **Forgot password?**.
3. Enter `john.doe` as the username and submit. <ProductName /> runs the recovery flow and sends a recovery email to `john.doe@example.com`.
4. Open the email in John's inbox and open the link inside.
5. <ProductName /> renders a **Set new password** screen. Enter a new password and submit.
6. <ProductName /> shows a confirmation and returns John to Wayfinder. Select **Sign in** again and sign in with the new password.

</details>
