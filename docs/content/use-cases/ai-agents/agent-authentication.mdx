# Secure Access for AI Agents

ThunderID enables secure, auditable access for AI agents—whether calling APIs, acting on behalf of users, or automating organization workflows. This page highlights what’s possible for agent-centric identity, now and in the future.

## At a Glance

<div className="uc-fact-grid uc-fact-grid--ai">
	<article className="uc-fact-card">
		<h3>Best Fit</h3>
		<p>Agents that call protected APIs or user-owned resources.</p>
	</article>
	<article className="uc-fact-card">
		<h3>Primary Goals</h3>
		<p>Tight scopes, delegated access, and auditable activity.</p>
	</article>
	<article className="uc-fact-card">
		<h3>Common Methods</h3>
		<p>Client credentials, authorization code, and short-lived access tokens.</p>
	</article>
	<article className="uc-fact-card">
		<h3>Watch For</h3>
		<p>Strict scope validation and predictable token rotation.</p>
	</article>
</div>

## What AI Agents Need

<ul className="uc-checklist">
	<li>Authenticate machine-to-machine agent calls.</li>
	<li>Support user-delegated actions from agent workflows.</li>
	<li>Scope tokens tightly to the minimum required permissions.</li>
	<li>Audit agent activity and token usage.</li>
</ul>

## Example Agent Journey

<ol className="uc-stage-list">
	<li><strong>Register identity:</strong> register each agent or agent platform as an application.</li>
	<li><strong>Handle background work:</strong> use client credentials for non-user operations.</li>
	<li><strong>Handle delegated work:</strong> use authorization code when user consent is required.</li>
	<li><strong>Constrain token lifetime:</strong> use short-lived access tokens and rotate secrets.</li>
	<li><strong>Guard every call:</strong> enforce scope and audience checks on all APIs.</li>
</ol>

## Key Considerations

<ol className="uc-action-list">
	<li>Define which agent actions require user delegation.</li>
	<li>Separate machine scopes from user scopes.</li>
	<li>Configure token lifetime and rotation policies.</li>
	<li>Add API-side scope and audience checks.</li>
	<li>Capture authentication and authorization events for auditing.</li>
</ol>
